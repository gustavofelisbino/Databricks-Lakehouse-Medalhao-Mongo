    Lakehouse Medalhão¶

    Pipeline completo Landing → Bronze → Silver → Gold construído sobre Databricks Free Edition + MongoDB Atlas, com Delta Lake, Unity Catalog e star schema Ralph Kimball para o domínio de seguros de automóvel.

    Começar agora Ver Arquitetura

    Camadas do Lakehouse¶

    • Landing

      Dump bruto do MongoDB Atlas em arquivos JSONL gravados em um Volume do Unity Catalog. Sem transformação — dado como veio.

      Saiba mais

    • Bronze

      Ingestão dos JSONL via spark.read.json para tabelas Delta Lake managed, com colunas de auditoria (data_hora_bronze, nome_arquivo).

      Saiba mais

    • Silver

      Aplicação de Data Quality: renome de colunas para padrão UPPER, trim, dedup e troca da auditoria bronze por auditoria silver.

      Saiba mais

    • Gold

      Star schema Kimball com 4 dimensões + 1 fato. Dimensões via MERGE (SCD Type 1); fato via TRUNCATE + INSERT.

      Saiba mais

    Stack Tecnológico¶

    • Databricks Free Edition

      Serverless Compute, Unity Catalog, Volumes e Jobs — tudo sem custos.

    • MongoDB Atlas M0

      Cluster gratuito na nuvem. Fonte dos 11 documentos do domínio de seguros.

    • Delta Lake

      Formato de tabela ACID em todas as camadas. Suporte a MERGE, time-travel e schema enforcement nativos.

    • PySpark + pymongo

      pymongo para extração (compatível com Serverless), PySpark para transformações e cargas em escala.

    O Job pipeline_seguradora_medalhao orquestra 5 tasks sequenciais no Databricks, cada uma executando um notebook em Serverless Compute:

    # Task Notebook Ação
    1 setup 00_setup_ambiente Cria schemas e volumes
    2 landing 01_landing_extracao_mongo Extrai MongoDB → JSONL
    3 bronze 02_bronze_ingestao JSONL → Delta Bronze
    4 silver 03_silver_data_quality Bronze → Silver (DQ)
    5 gold 04_gold_dimensional Silver → Gold (star schema)

    Sobre o Projeto¶

    Contexto Acadêmico

    Este projeto é o Trabalho 3 de uma disciplina de Engenharia de Dados. O objetivo é demonstrar a arquitetura Medallion em uma plataforma cloud moderna, partindo de uma fonte não-relacional (MongoDB) e entregando dados analíticos prontos para consumo no Gold.

    Domínio: Seguros de automóvel · 11 collections · 5 tabelas Gold
